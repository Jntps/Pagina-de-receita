![Prévia do design para o desafio de codificação da página de receita](./preview.jpg)

## Bem-vindo! 👋

Obrigado por conferir este desafio de codificação front-end.

Os desafios do [Frontend Mentor](https://www.frontendmentor.io) ajudam você a aprimorar suas habilidades de programação criando projetos realistas.

**Para realizar este desafio, você precisa de conhecimentos básicos de HTML e CSS.**

## O desafio

Seu desafio é criar esta página de receita e deixá-la o mais fiel possível ao design original.

Você pode usar quaisquer ferramentas que desejar para concluir o desafio. Portanto, se houver algo que você queira praticar, sinta-se à vontade para experimentar.

### Precisa de ajuda com o desafio?

[Entre na nossa comunidade](https://www.frontendmentor.io/community) e tire suas dúvidas no canal **#help**.

## Onde encontrar tudo

Sua tarefa é desenvolver o projeto seguindo os designs presentes na pasta `/design`. Você encontrará versões do design tanto para dispositivos móveis (mobile) quanto para desktop.

Os designs estão em formato estático JPG. O uso de JPGs significa que você precisará usar seu próprio julgamento para definir estilos como `font-size`, `padding` e `margin`.

Se você quiser o arquivo de design do Figma para ganhar experiência com ferramentas profissionais e criar projetos mais precisos com mais rapidez, pode [assinar como membro PRO](https://www.frontendmentor.io/pro).

Todos os recursos (assets) necessários para este projeto estão na pasta `/assets`. As imagens já estão exportadas no tamanho de tela correto e otimizadas.

Também incluímos arquivos de fontes variáveis ​​e estáticas para as fontes necessárias neste projeto. Você pode optar por vincular ao Google Fonts ou usar os arquivos de fonte locais para hospedar as fontes por conta própria. Observe que removemos os arquivos de fontes estáticas para os pesos de fonte que não são necessários neste projeto.

Há também um arquivo `style-guide.md` contendo as informações de que você precisará, como paleta de cores e fontes. ## Uso de assistentes de programação com IA

Incluímos dois arquivos para ajudar você caso utilize assistentes de programação com IA (como Claude, GitHub Copilot, Cursor, etc.) ao realizar este desafio:

- `AGENTS.md` - Contém instruções detalhadas para assistentes de IA sobre como ajudar você neste desafio. O conteúdo é adaptado ao nível de dificuldade do desafio; assim, a IA fornecerá orientações adequadas ao seu estágio de aprendizado — oferecendo mais suporte em desafios para iniciantes e incentivando maior autonomia em desafios avançados.
- `CLAUDE.md` - Um arquivo de referência que direciona ferramentas baseadas no Claude para as instruções contidas em `AGENTS.md`.

**Como utilizá-los:** Você não precisa fazer nada! Esses arquivos são detectados automaticamente pela maioria das ferramentas de programação com IA. A IA os lerá e ajustará seu comportamento para atuar como um parceiro de aprendizado eficaz, guiando você em direção às soluções em vez de simplesmente fornecer as respostas.

**Observação:** Estes arquivos foram criados para ajudar você a *aprender*, e não para realizar o trabalho em seu lugar. A IA está instruída a fazer perguntas, dar dicas e explicar conceitos, em vez de escrever soluções completas.

## Desenvolvendo o projeto

Sinta-se à vontade para utilizar qualquer fluxo de trabalho com o qual se sinta confortável. Abaixo, apresentamos um processo sugerido, mas não é obrigatório seguir estes passos:

1. Inicialize seu projeto como um repositório público no [GitHub](https://github.com/). Criar um repositório facilitará o compartilhamento do seu código com a comunidade caso precise de ajuda. Se não souber como fazer isso, [consulte este recurso do Try Git](https://try.github.io/).
2. Configure seu repositório para publicar o código em um endereço web. Isso também será útil caso precise de ajuda durante o desafio, pois você poderá compartilhar a URL do projeto junto com a URL do repositório. Existem várias maneiras de fazer isso; apresentamos algumas recomendações abaixo.
3. Analise os designs para começar a planejar como você abordará o projeto. Esta etapa é fundamental para ajudar a antecipar quais classes CSS serão necessárias para criar estilos reutilizáveis.
4. Antes de adicionar estilos, estruture o conteúdo com HTML. Escrever o HTML primeiro ajuda a manter o foco na criação de um conteúdo bem estruturado. 5. Escreva os estilos base do seu projeto, incluindo estilos gerais de conteúdo, como `font-family` e `font-size`.
6. Comece adicionando estilos do topo da página para baixo. Só passe para a próxima seção quando estiver satisfeito com a conclusão da área em que está trabalhando.

## Publicando seu projeto

Como mencionado acima, existem várias maneiras de hospedar seu projeto gratuitamente. Nossas recomendações de hospedagem são:

- [GitHub Pages](https://pages.github.com/)
- [Vercel](https://vercel.com/)
- [Netlify](https://www.netlify.com/)

Você pode hospedar seu site usando uma dessas soluções ou qualquer um de nossos outros provedores confiáveis. [Leia mais sobre nossas recomendações de hospedagem confiável](https://www.frontendmentor.io/guides/hosting-your-solution).

## Crie um `README.md` personalizado

Recomendamos fortemente substituir este `README.md` por um personalizado. Fornecemos um modelo no arquivo [`README-template.md`](./README-template.md) dentro deste código inicial.

O modelo serve como guia sobre o que incluir. O arquivo `README` ajudará você a explicar seu projeto e a refletir sobre o que aprendeu. Sinta-se à vontade para editar nosso modelo como quiser.

Depois de adicionar suas informações ao modelo, exclua este arquivo e renomeie o arquivo `README-template.md` para `README.md`. Isso fará com que ele apareça como o arquivo README do seu repositório.

Envie sua solução na plataforma para que o restante da comunidade possa vê-la. Siga nosso ["Guia completo para envio de soluções"](https://www.frontendmentor.io/guides/how-to-submit-solutions) para obter dicas sobre como fazer isso.

## Compartilhando sua solução

Existem vários lugares onde você pode compartilhar sua solução:

1. Compartilhe a página da sua solução no canal **#finished-projects** da nossa [comunidade](https://www.frontendmentor.io/community).
2. Compartilhe no [X (antigo Twitter)](https://x.com/frontendmentor) e mencione **@frontendmentor**, incluindo os links do repositório e da versão online (live URL) na sua publicação. Adoraríamos ver o que você construiu e ajudar a divulgar.
3. Compartilhe sua solução no [LinkedIn](https://www.linkedin.com/company/frontend-mentor/).
4. Escreva um post no blog sobre a experiência de criar seu projeto. Escrever sobre seu fluxo de trabalho e escolhas técnicas, além de explicar seu código, é uma maneira excelente de reforçar o que você aprendeu. Ótimas plataformas para escrever são [dev.to](https://dev.to/), [Hashnode](https://hashnode.com/) e [CodeNewbie](https://community.codenewbie.org/).

Disponibilizamos modelos para ajudar você a compartilhar sua solução após enviá-la na plataforma. Sinta-se à vontade para editá-los e incluir perguntas específicas quando estiver buscando feedback.

Quanto mais específicas forem suas perguntas, maior a probabilidade de outro membro da comunidade lhe dar um feedback.
